# Contributing to my-research

Thank you for your interest in contributing! Please follow the guidelines below to keep the repository organized and consistent.

## How to Contribute

1. **Fork** the repository and create a new branch from `master`:
   ```bash
   git checkout -b feature/your-topic
   ```

2. **Make your changes** following the folder conventions below.

3. **Commit** with a clear, descriptive message:
   ```bash
   git commit -m "Add summary of XYZ paper to paper1/"
   ```

4. **Push** your branch and open a **Pull Request** against `master`.

## Folder Conventions

| Folder | What belongs here |
|---|---|
| `paper1/` | Drafts, notes, and figures for the first paper |
| `paper2/` | Drafts, notes, and figures for the second paper |
| `data/` | Raw and processed datasets; include a brief description of each file |
| `references/` | `.bib` files, citation lists, PDFs of referenced works |

## File Naming

- Use lowercase with hyphens: `my-dataset.csv`, `draft-v2.tex`
- Avoid spaces and special characters in filenames
- Version drafts explicitly: `draft-v1.md`, `draft-v2.md`

## Commit Message Style

- Use present tense: `Add ...`, `Update ...`, `Fix ...`
- Keep the first line under 72 characters
- Reference related issues or papers where relevant

## Reporting Issues

Open a [GitHub Issue](https://github.com/kraghu356/my-research/issues) to:
- Report a broken file or link
- Suggest a new folder or structural change
- Ask a question about the research content
